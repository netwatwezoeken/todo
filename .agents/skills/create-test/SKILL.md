## Test autonomy

Use gherkin to specify a behavioral test.
Gherkin specifcations can be found in the `Specifications` project.

The implementation of gherkin is realized by using reqnroll. Gherkin phrases are bound to Bindings in the `AspireIntegrationTest` project. For example `LoginPage.cs`.
Classes that implement Binding for use with the UI are structured to match pages in the web UI. For example: `LoginPage.cs` and `TodoPage.cs`. Newly added Bindings must follow that same structure.

In these page-oriented Bindings Playwright is used to instruct and assert the web UI. When adding testcode use the `playwright-cli` to explore the UI, based on those findings create the appropriate actions in Binding classes.

Any scenario involving the UI should always be tagged with `@integration` in the feature file.
The `Specifications` project only includes specifications. Bindings are to be created in the `AspireIntegrationTest` project.
Never create any `*.feature.cs` file, these are generated by reqnroll.

## Example code

Example feature file `user.feature`:
```gherkindotnet
Feature: create user

@integration 
Scenario: A user adds a todo
    Given user navigates to the main page
    When they fill in 'jos@netwatwezoeken.nl' as email
    And they fill in password 'T0psecret!'
    And they click Create
    Then user is logged in
```

Example binding classes that are used by thin feature file:

`LoginPage.cs`
```csharp
[Binding]
public class LoginPage(ITestOutputHelper output): BaseStepDefinition(output)
{
    [Given("user navigates to the main page")]
    public async Task GoToLoginPage()
    {
        await Page.GotoAsync(Fixture.WebUri.ToString());
    }

    [When("they fill in {string} as email")]
    public async Task FillInAsEmail(string email)
    {
        await Page.GetByRole(AriaRole.Textbox, new() { Name = "Email" }).FillAsync(email);
    }

    [When("they fill in password {string}")]
    public async Task FillInPassword(string password)
    {
        await Page.GetByRole(AriaRole.Textbox, new() { Name = "Password" }).FillAsync(password);
    }
    
    [When("they click Create")]
    public async Task ClickCreate()
    {
        await Page.GetByRole(AriaRole.Button, new() { Name = "Create User" }).ClickAsync();
    }
}
```

`TodoPage.cs`
```csharp
[Binding]
public class TodoPage(ITestOutputHelper output): BaseStepDefinition(output)
{
    [Then("user is logged in")]
    public async Task LoggedIn()
    {
        await Assertions.Expect(Page.GetByRole(AriaRole.Button, new() { Name = "Logout" })).ToBeVisibleAsync();
    }
}
```

## Running tests

Run these tests by using `dotnet test` on the `Specifications` Project.
Before doing so ensure that no other aspire instance is running by running `aspire ps` and the stop any runnning instance by running `aspire stop` (see `aspire-cli` skill for detailed instructions)
